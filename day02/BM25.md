### 一、 为什么有了向量检索，还要学 BM25？

向量检索（Embedding）虽然厉害，但它有个致命的弱点：**它太“自以为是”了。**

*   **场景：** 用户搜一个具体的零件编号“SN-9527”，或者一个生僻的药名。
*   **向量检索的下场：** 它会觉得“SN-9528”或者“SN-9526”在语义空间里离得很近，于是给你搜出一堆错误的信息。
*   **BM25 的表现：** 它会死死盯着“9527”这几个数字，精准地把包含这个编号的文档抓出来。

**记住导师的话：** 向量检索管“语义相似”，BM25 管“词汇匹配”。在大厂做 Agent，你如果只用向量检索，你的系统稳定性就是个笑话。

### 二、 核心原理：它是怎么算分的？

BM25 是 TF-IDF 的进化版。你不需要背那个复杂的公式，但你必须理解它的三个核心维度（面试必考）：

#### 1. TF (Term Frequency) —— 词频饱和度
*   **理论：** 一个词在文档里出现次数越多，文档越相关。
*   **BM25 的改进：** 它是非线性的。在 TF-IDF 里，一个词出现 100 次的权重是一个词出现 1 次的 100 倍。但在 BM25 里，它认为一个词出现 5 次后，再出现第 6 次、第 100 次，贡献度就不怎么增加了。这叫**词频饱和**。
*   **参数 $k_1$：** 控制饱和的速度。$k_1$ 越小，饱和越快。

#### 2. IDF (Inverse Document Frequency) —— 逆文档频率
*   **理论：** 物以稀为贵。
*   **逻辑：** 在一万篇文章里，“的”出现了九千次，它就一文不值；但“华师大”只出现了十次，那它就是核心关键词。BM25 会给稀有词极高的权重。

#### 3. Document Length Normalization —— 文档长度归一化
*   **理论：** 短文里出现一次关键词，比长篇大论里出现一次关键词更有含金量。
*   **参数 $b$：** 惩罚项。如果 $b$ 越大，对长文本的惩罚就越狠。

### 三、 工业级实现：怎么在 Python 里玩转它？

你昨天已经装了 `rank_bm25`，现在我教你如何在 Agent 中正确地封装它。大厂不需要你手写公式，但需要你把检索流程工程化。

```python
import jieba
from rank_bm25 import BM25Okapi

class BM25Retriever:
    def __init__(self, corpus: list):
        # 提醒：中文检索必须分词！不分词的 BM25 就是垃圾。
        self.tokenized_corpus = [list(jieba.cut(doc)) for doc in corpus]
        self.bm25 = BM25Okapi(self.tokenized_corpus)
        self.raw_corpus = corpus

    def search(self, query: str, top_n: int = 3):
        # 1. 对查询词进行分词
        tokenized_query = list(jieba.cut(query))
        
        # 2. 获取分值 (每一个文档针对这个 query 的得分)
        scores = self.bm25.get_scores(tokenized_query)
        
        # 3. 排序并返回原文
        top_docs = self.bm25.get_top_n(tokenized_query, self.raw_corpus, n=top_n)
        return top_docs

# 使用示例
corpus = [
    "计算机科学与技术学院欢迎你。",
    "深度学习是人工智能的一个重要分支。",
    "Agent 是大语言模型的核心应用场景。"
]

retriever = BM25Retriever(corpus)
res = retriever.search("计算机")
print(res)
```

### 四、 杀手锏：RRF（Reciprocal Rank Fusion）

既然你现在既会 ChromaDB（向量检索），又会 BM25（关键词检索），大厂面试官一定会问你：**“你怎么把这两者的结果合并？”**

你不能简单地把分数加起来，因为向量检索的分数（0~1）和 BM25 的分数（可能是几十）不在一个量级。

你要用 **RRF 算法**。它的逻辑极其粗暴但有效：**看排名，不看分数。**

*   **公式思想：** $Score = \frac{1}{k + rank_{vector}} + \frac{1}{k + rank_{bm25}}$
*   如果一个文档在向量检索里排第 1，在 BM25 里也排第 1，那它就是绝对的 Top 1。
